    **Dexie Database (src/client/src/services/):**
    1. Create db.ts: Dexie.js database schema per RESEARCH.md Pattern 4.
       - MixGodDB extends Dexie with tables: projects, tracks, playlists
       - ProjectRecord: id (auto), name, createdAt, updatedAt
       - TrackRecord: id (auto), projectId, serverId, title, artist, bpm, bpmRaw, bpmCorrected, key, keyConfidence, energy, genrePrimary, genreSecondary, genreConfidence, duration, format, bitrate, sampleRate, dateAdded, filename, peaksUrl, analysisStatus, analysisConfidence, userOverrides
       - SmartPlaylistRecord: id (auto), projectId, name, filters
       - Indexes: tracks by projectId, serverId; playlists by projectId
       - Export singleton: `export const db = new MixGodDB()`
       - Export function: exportProject(projectId) -> JSON string (project + tracks + playlists). This is the .mixgod file format. Keep lightweight (~50KB, metadata only, no audio data).
       - Export function: importProject(json: string) -> creates project + tracks + playlists from JSON. Server IDs need to be re-resolved if importing on different machine (deferred -- Phase 1 assumes same machine).

    **Project Store (src/client/src/stores/):**
    2. Create projectStore.ts (Zustand):
       - State: currentProjectId: number | null, projects: ProjectRecord[], isLoading: boolean
       - Actions: createProject(name), loadProject(id), loadProjects(), deleteProject(id), exportProject(), importProject(file)
       - On createProject: add to Dexie, set as current, clear libraryStore tracks
       - On loadProject: load from Dexie, populate libraryStore with tracks from Dexie
       - Auto-save: subscribe to libraryStore changes, debounce 2 seconds, write current tracks to Dexie. Use Zustand subscribe() + setTimeout debounce.
       - On app startup: load projects from Dexie. If only one project, auto-load it. If multiple, show project switcher. If none, create "Untitled Mix" project.

       - Neon-dark styled modal with backdrop blur

    4. Create ExportDialog.tsx: Export current project as .mixgod file.

       - Filename: `{projectName}.mixgod`
