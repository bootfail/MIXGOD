5. Create DownloadService.cs -- mirror AnalysisService pattern:
   - Constructor: inject ILogger, IConfiguration. Read "Download:YtDlpPath" config (default "yt-dlp"), "Download:TimeoutSeconds" (default 300).
   - `IsAvailable()`: Spawn `yt-dlp --version`, return true if exit 0, false otherwise. Cache result.
   - `GetMetadataAsync(url)`: Spawn `yt-dlp --dump-json --no-download "{url}"`. Read stdout as JSON. Parse fields: title, uploader (as artist), thumbnail, duration, id, webpage_url. Apply title parsing: strip common suffixes like "(Official Video)", "(Lyric Video)", "[HD]", "(Audio)" using regex. Split "Artist - Title" format using dash separator regex (handle en-dash and em-dash). Set `StandardOutputEncoding = Encoding.UTF8` and `StandardErrorEncoding = Encoding.UTF8` per Windows encoding pitfall. Return DownloadResult with parsed metadata.
   - `DownloadAudioAsync(url, outputDir, onProgress)`: Spawn yt-dlp with args: `-x --audio-format mp3 --audio-quality 0 --newline --progress-template "download:PROGRESS|%(progress._percent_str)s|%(progress._speed_str)s|%(progress._eta_str)s" --write-thumbnail -o "{outputDir}/audio.%(ext)s" "{url}"`. Read stdout line-by-line. Parse lines starting with "download:PROGRESS|" to extract percentage (strip whitespace and %). Call onProgress(percent, eta) for each parsed line. On non-zero exit, throw with stderr message. Return path to downloaded audio file (find mp3 file in outputDir).
   - Title parsing helper (private static): `ParseTitle(string rawTitle)` returns `(string title, string artist)`. Regex strips suffixes, then splits on " - " (or en/em dash).

   - Extends BackgroundService. Inject: ChannelReader&lt;DownloadJob&gt;, ITrackStore, IDownloadService, IAudioStorageService, ChannelWriter&lt;AnalysisJob&gt;, ILogger.
   - SemaphoreSlim(2, 2) -- allow 2 concurrent downloads (bandwidth constraint, per research).
   - ExecuteAsync: ReadAllAsync loop, semaphore-bounded concurrency via Task.Run (same pattern as AnalysisQueueProcessor).
   - ProcessJobAsync(DownloadJob job):
     a. Update track: DownloadStatus = Downloading
     b. Create output directory: audio-storage/{trackId}/
     c. Call _downloadService.GetMetadataAsync(job.Url) -- update track with title, artist from parsed metadata
     d. Call _downloadService.DownloadAudioAsync(job.Url, outputDir, onProgress) where onProgress stores progress in a static ConcurrentDictionary&lt;string, (int Percent, string Eta)&gt; (volatile progress data, not in Track model per research anti-pattern guidance)
     e. On success: Update track FilePath, DownloadStatus=Done, Filename, Format="mp3". Find and set ThumbnailPath if thumbnail file exists in outputDir.
     f. Queue AnalysisJob to existing analysis channel: `await _analysisQueue.WriteAsync(new AnalysisJob(job.TrackId, filePath))`
     g. On error: Update track DownloadStatus=Error, ErrorMessage with meaningful message. Parse yt-dlp stderr for known patterns ("Video unavailable", "Private video", "not available in your country") and provide user-friendly messages.
   - Expose static method `GetProgress(string trackId)` to read from the ConcurrentDictionary (used by status endpoint).
