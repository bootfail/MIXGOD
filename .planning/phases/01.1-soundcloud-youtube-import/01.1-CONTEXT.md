# Phase 01.1: SoundCloud/YouTube Import - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Users can paste YouTube or SoundCloud URLs and get the audio downloaded into their track library. Downloaded tracks enter the same analysis pipeline as uploaded files (BPM/key/energy/genre). No in-app search, no playlist URL expansion, no additional platforms — those remain in Phase 4.

**Requirements:** IMPORT-02 (YouTube/SoundCloud URL download via yt-dlp)

</domain>

<decisions>
## Implementation Decisions

### URL Input UX
- Dedicated "Import from URL" dialog, triggered by a button next to the existing upload/browse button
- Dialog contains a multi-line text area — paste one URL per line for batch import
- Live URL validation as user types/pastes: green check for valid YouTube/SoundCloud URLs, red X for invalid
- Import button disabled until at least one valid URL is present

### Download Feedback
- Track appears in the library table immediately with status: downloading → queued → analyzing → done (extends existing analysis status flow)
- Download progress shown inline per row: percentage + ETA (e.g., "Downloading... 65% — ~30s remaining")
- Global progress bar at top for batch imports, reusing the same pattern as batch analysis: "Downloading tracks... 3/10"
- Download failures show error icon in the row with error message (e.g., "Video unavailable", "Region locked") + retry button. Same pattern as analysis errors from Phase 1

### Metadata Extraction
- Smart parse of video/track titles: split "Artist - Title (Official Video)" format, strip common suffixes like "(Official Video)", "(Lyric Video)", "[HD]", "(Extended Mix)" kept
- Pre-fill Title and Artist fields automatically; user can always edit
- Download and store thumbnail images from YouTube/SoundCloud — display as album art column in track table and in player bar
- Best available audio quality downloaded (highest quality stream from yt-dlp)
- Store and display original source URL as metadata — show platform icon (YouTube/SoundCloud logo) in library, click opens original page

### Platform Scope
- YouTube + SoundCloud only — no other platforms in this phase
- URL paste only — no in-app search (SC-01 stays in Phase 4)
- Duplicate URL detection: warn if URL already imported, but allow re-import if user chooses
- yt-dlp runs server-side on the .NET backend — frontend sends URLs, backend spawns yt-dlp, streams progress via API

### Claude's Discretion
- Exact dialog layout and neon styling (consistent with existing theme)
- yt-dlp output parsing and progress extraction approach
- How to handle yt-dlp not being installed (error message, install instructions)
- Exact title parsing regex patterns and edge case handling
- Thumbnail storage format and sizing

</decisions>

<specifics>
## Specific Ideas

- This phase was inserted as urgent because user has no local audio files — URL import is needed to test any functionality built in Phase 1
- The import flow should feel as seamless as file upload: paste URLs → tracks appear in library → analysis runs automatically
- Source icons (YouTube/SoundCloud logos) add visual context and let user trace back to original source

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `TracksController.cs`: Upload endpoint pattern (store file → create track → queue analysis) — URL import follows same flow, just downloads first
- `DropZone.tsx` + `UploadProgress.tsx`: Existing upload UI components — import button sits alongside these
- `AnalysisService.cs` + analysis queue (Channel<AnalysisJob>): Same pipeline processes downloaded tracks
- `libraryStore.ts`: Track state management — extend with download status
- `analysisStore.ts`: Analysis polling with exponential backoff — reuse for download + analysis status
- `AudioStorageService.cs`: File storage — downloaded files go to same location as uploaded files
- `PeakService.cs`: Waveform peak generation — runs on downloaded files identically

### Established Patterns
- ConcurrentDictionary in-memory storage (no DB in Phase 1)
- Analysis status polling with setInterval + exponential backoff (2s→10s)
- Track model with full metadata fields (BPM, key, energy, genre, etc.)
- Neon-dark theme with multicolor accents and glow effects

### Integration Points
- New endpoint on TracksController (POST /api/tracks/import) accepts URLs instead of files
- Backend needs yt-dlp as a system dependency
- Track model needs new fields: sourceUrl, sourceType (youtube/soundcloud/upload), thumbnailUrl
- Download status extends existing AnalysisStatus enum or adds parallel DownloadStatus

</code_context>
